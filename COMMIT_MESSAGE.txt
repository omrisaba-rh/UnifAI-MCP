feat: Add error handling, caching, and security improvements

This commit implements three major improvements to the UnifAI MCP server:

1. Error Handling & Resilience:
   - Add 5-minute timeout protection for workflow execution
   - Enhance error handling with better retry logic
   - Add detailed logging for debugging
   - Provide clear error messages with session IDs for recovery

2. Progress Reporting:
   - Add real-time progress updates every 30 seconds
   - Show elapsed time during workflow execution
   - Improve user visibility into long-running workflows

3. Blueprint Caching:
   - Implement in-memory cache with 5-minute TTL
   - Reduce API calls to UnifAI backend
   - Add cache control with use_cache parameter
   - Include debug logging for cache hits/misses

4. Security Improvements:
   - Enable SSL verification by default
   - Add VERIFY_SSL configuration option
   - Log warnings when SSL verification is disabled
   - Update documentation for secure configuration

Breaking Changes:
- SSL verification now enabled by default. Set VERIFY_SSL=false
  in .env for development environments with self-signed certificates.

Files Modified:
- src/unifai_mcp/server.py: Enhanced run_workflow with timeout and progress
- src/unifai_mcp/unifai_client.py: Added caching layer for blueprints
- src/unifai_mcp/config.py: Added verify_ssl setting
- .env.example: Added VERIFY_SSL configuration
- README.md: Updated features and configuration sections

Files Added:
- IMPROVEMENTS.md: Detailed documentation of all improvements
- CHANGELOG.md: Version history and change log
